Create the Database & Tables

In pgAdmin 4: right-click Databases → Create → Database and name it library_db. Then open the Query Tool and run each CREATE TABLE below.

💡Create tables in this order: books and members first, then borrowings — because borrowings references both via FOREIGN KEY.

Step 1 — books Table

Stores every book available in the library.

SQL 
CREATE TABLE books (
    book_id          SERIAL        PRIMARY KEY,
    title            VARCHAR(200)  NOT NULL,
    author           VARCHAR(100)  NOT NULL,
    isbn             VARCHAR(20)   UNIQUE NOT NULL,
    publication_year INTEGER       CHECK (publication_year > 1000),
    available_copies INTEGER       DEFAULT 1
                                   CHECK (available_copies >= 0)
);
Column Constraint Purpose
book_id SERIAL PRIMARY KEY Auto-generated unique ID for each book
isbn UNIQUE NOT NULL Each book has a unique ISBN; must be provided
publication_year CHECK > 1000 Rejects obviously wrong year values
available_copies CHECK >= 0 Stock can never be negative

Step 2 — members Table

Stores library members who are allowed to borrow books.

SQL 
CREATE TABLE members (
    member_id  SERIAL       PRIMARY KEY,
    name       VARCHAR(100) NOT NULL,
    email      VARCHAR(100) UNIQUE NOT NULL,
    phone      VARCHAR(20),
    join_date  DATE         DEFAULT CURRENT_DATE
);

Step 3 — borrowings Table

Records every borrowing transaction. A NULL in return_date means the book is still borrowed.

SQL 
CREATE TABLE borrowings (
    borrowing_id  SERIAL   PRIMARY KEY,
    member_id     INTEGER  NOT NULL,
    book_id       INTEGER  NOT NULL,
    borrow_date   DATE     DEFAULT CURRENT_DATE,
    due_date      DATE     NOT NULL,
    return_date   DATE,    -- NULL = still borrowed
    FOREIGN KEY (member_id) REFERENCES members(member_id)
        ON DELETE CASCADE,
    FOREIGN KEY (book_id)   REFERENCES books(book_id)
        ON DELETE CASCADE
);

Required Queries

Write and run each query in pgAdmin. Attach results or a screenshot with your submission.

Query 1 🔍 List All Currently Borrowed Books

Show books that have not been returned yet. Include member name, book title, borrow date, and due date.

SQL 
SELECT
    m.name         AS member_name,
    b.title        AS book_title,
    br.borrow_date,
    br.due_date
FROM borrowings br
    INNER JOIN members m ON br.member_id = m.member_id
    INNER JOIN books   b ON br.book_id   = b.book_id
WHERE  br.return_date IS NULL
ORDER  BY br.due_date;
💡Always use IS NULL — not = NULL. The = operator never matches NULL values in SQL.
Query 2 🔍 Find Overdue Books

A book is overdue if it is still borrowed and its due_date has already passed. Use CURRENT_DATE for today's date.

SQL 
SELECT
    m.name                      AS member_name,
    b.title                     AS book_title,
    br.due_date,
    CURRENT_DATE - br.due_date  AS days_overdue
FROM borrowings br
    INNER JOIN members m ON br.member_id = m.member_id
    INNER JOIN books   b ON br.book_id   = b.book_id
WHERE  br.return_date IS NULL
  AND  br.due_date < CURRENT_DATE
ORDER  BY days_overdue DESC;
Query 3 🔍 Members With No Borrowings

Use LEFT JOIN so members who never borrowed still appear. Then filter where no borrowing was found.

SQL 
SELECT
    m.member_id,
    m.name,
    m.email
FROM members m
    LEFT JOIN borrowings br ON m.member_id = br.member_id
WHERE  br.borrowing_id IS NULL
ORDER  BY m.name;
💡LEFT JOIN keeps all rows from members, even with no match in borrowings. WHERE borrowing_id IS NULL then keeps only those with no match.
Query 4 🔍 Total Borrowings Per Member

Count how many times each member has borrowed. Include members with zero borrowings.

SQL 
SELECT
    m.name,
    COUNT(br.borrowing_id)  AS total_borrowings
FROM members m
    LEFT JOIN borrowings br ON m.member_id = br.member_id
GROUP  BY m.member_id, m.name
ORDER  BY total_borrowings DESC;
Query 5 🔍 Most Popular Books

Rank every book by borrow count. Books never borrowed should show 0.

SQL 
SELECT
    b.title,
    b.author,
    COUNT(br.borrowing_id)  AS times_borrowed
FROM books b
    LEFT JOIN borrowings br ON b.book_id = br.book_id
GROUP  BY b.book_id, b.title, b.author
ORDER  BY times_borrowed DESC;
